Overview: The API Journey

Laravel's API capabilities have evolved dramatically since version 5.8. From basic token guards and Passport-only OAuth, to the lightweight Sanctum, to the streamlined opt-in structure of Laravel 11/12 — each version brought meaningful improvements to how developers build and secure APIs.

v5.8 (Feb 2019) — SHA-256 Token Hashing

Basic token guard updated to support hashed token storage for improved security.

v6.0 (Sep 2019) — Semantic Versioning, Vapor Support, Lazy Collections

Framework aligned to semver; lazy collections improved large API dataset handling. First LTS after 5.x.

v7.0 (Mar 2020) — Sanctum, HTTP Client, Route Caching

Laravel Sanctum debuted as a lightweight SPA/token auth alternative to Passport. New Http:: facade for consuming external APIs.

v8.0 (Sep 2020) — Model Factories, Job Batching, Improved Rate Limiting

Massive improvements to testing and background processing patterns for APIs. Dynamic rate limiting per user.

v9.0 (Feb 2022) — Symfony 6 Components, Scoped Bindings, Enum Support

Native PHP 8.1 enum support in routes and scoped nested resource bindings.

v10.0 (Feb 2023) — PHP 8.1 Minimum, Pennant Feature Flags

Feature flags with Pennant for API feature rollouts; full PHP 8.1 native types throughout.

v11.0 (Mar 2024) — Slim Skeleton, Opt-in API, Per-Second Rate Limiting

Biggest structural change: api.php removed by default, installed on demand. Centralized bootstrap/app.php.

v12.0 (Feb 2025) — Starter Kits, WorkOS AuthKit, Stability Release

New React/Vue/Livewire starter kits with modern auth including passkeys and SSO support.

v5.8

Laravel 5.8 — API Security Foundations

February 2019
🔒

SHA-256 Token Hashing

The built-in token guard now supports storing API tokens as SHA-256 hashes instead of plain text, significantly improving security for simple token-based APIs.

Security Improvement
📧

Improved Email Validation

Adopted the egulias/email-validator package, fixing edge cases in API input validation where valid international email addresses were rejected.

Improved
🔗

Eloquent hasOneThrough

New relationship type allowing navigation through intermediate models in a single query — useful for deeply nested API resource relationships.

New
📋

Token Guard Still Basic

The simple token guard remained the only built-in lightweight option. Passport was the recommended solution for production APIs. Sanctum would not arrive until v7.0.

Context
config/auth.php — SHA-256 Token Hashing (v5.8)
// Enable hashed token storage in your guard config
'api' => [
    'driver'   => 'token',
    'provider' => 'users',
    'hash'     => true, // NEW in v5.8 - stores SHA-256 hash
],

// When creating a token:
$token = Str::random(60);
$user->forceFill([
    'api_token' => hash('sha256', $token),
])->save();

// Return the plain token to the client (hash is stored in DB)
return response()->json(['token' => $token]);
Recommendation in v5.8: The built-in token guard was always meant for simple use cases. Laravel already recommended Passport (OAuth2) for production APIs. Sanctum arrives in v7.0 as the ideal middle ground.

App\Policies\PostPolicy.php — Richer API Auth Responses (v6.0)
use Illuminate\Auth\Access\Response;

public function update(User $user, Post $post): Response
{
    return $user->id === $post->user_id
        ? Response::allow()
        : Response::deny('You do not own this post.');
    // API clients now receive: {"message": "You do not own this post."} with 403
}
Api/ExportController.php — Lazy Collections for large API datasets (v6.0)
use App\Models\Post;

// Stream 100,000+ records without memory explosion
public function export()
{
    return response()->streamDownload(function () {
        // cursor() returns a LazyCollection - one record at a time
        Post::cursor()->each(function ($post) {
            echo json_encode($post) . "\n";
        });
    }, 'posts.jsonl');
}
v7.0

Laravel 7.0 — Sanctum Born, HTTP Client Arrives

March 2020
Major API Release: Laravel 7 introduced Sanctum — a lightweight authentication package that became the go-to solution for SPA and mobile API authentication, replacing the need for full OAuth2 via Passport in most cases.
🛡️

Laravel Sanctum

A featherweight authentication system for SPAs, mobile apps, and token-based APIs. Supports both cookie-based session auth and personal API tokens without OAuth2 overhead.

New Package
🌐

HTTP Client (Http:: Facade)

New first-party Http facade wrapping Guzzle. Consuming external APIs from Laravel is now clean and testable with built-in fake/mock support — no more raw Guzzle boilerplate.

New
📝

Fluent String Operations

New Str::of() object-oriented string manipulation, useful for transforming API response data in a clean, chainable way.

New
🔄

Route Model Binding Custom Resolution

Custom resolution logic for route model bindings, allowing APIs to resolve models by any attribute (slug, uuid, etc.), not just the primary key id.

Improved
routes/api.php — Sanctum Authentication Setup (v7.0)
// 1. Install: composer require laravel/sanctum
// 2. Add HasApiTokens trait to User model
use Laravel\Sanctum\HasApiTokens;

class User extends Authenticatable
{
    use HasApiTokens, HasFactory, Notifiable;
}

// 3. Issue a scoped token on login
Route::post('/login', function (Request $request) {
    // ... validate credentials ...
    // Create token with abilities (scopes)
    $token = $request->user()->createToken('mobile-app', ['posts:read', 'posts:write']);
    return response()->json(['token' => $token->plainTextToken]);
});

// 4. Protect routes with Sanctum middleware
Route::middleware('auth:sanctum')->group(function () {
    Route::get('/user', fn (Request $r) => $r->user());
    Route::apiResource('posts', PostController::class);
});

// 5. Check token abilities in controllers
if (! $request->user()->tokenCan('posts:write')) {
    return response()->json(['error' => 'Insufficient token scope'], 403);
}
App\Services\ — Fluent HTTP Client for external APIs (v7.0)
use Illuminate\Support\Facades\Http;

// Old way: raw Guzzle, verbose and hard to test
// New way: clean, fluent, fully testable
$response = Http::withToken($apiToken)
    ->retry(3, 100)          // Retry 3 times, 100ms delay
    ->timeout(10)            // 10 second timeout
    ->acceptJson()
    ->get('https://api.example.com/users', [
        'page'  => 1,
        'limit' => 50,
    ]);

if ($response->successful()) {
    return $response->json();
}
$response->throw(); // Throws HttpClientException on error

// In tests:
Http::fake(['api.example.com/*' => Http::response(['data' => []])]);
v8.0

Laravel 8.0 — Factories, Job Batching & Dynamic Rate Limiting

September 2020
🏭

Class-Based Model Factories

Factories moved from functions to dedicated classes with HasFactory trait. Makes API testing far more expressive with factory states, sequences, and relationships.

Redesigned
📦

Job Batching

Bus::batch() allows dispatching groups of API-processing jobs and tracking collective completion, progress, and failure with callbacks.

New
🚦

Dynamic Rate Limiting

Rate limits dynamically computed per user. Premium users get higher limits, free users get lower — all configured elegantly without custom middleware.

Improved
🎨

Route Controller Groups

New Route::controller() groups routes by controller class, dramatically reducing repetition in large api.php files.

New
🧩

Invokable Validation Rules

Custom API validation rules can now be simple invokable classes instead of requiring passes() and message() methods separately.

New
🔧

Maintenance Mode Bypass

Maintenance mode supports bypass tokens, so API clients with a secret key can continue operating during maintenance windows.

Improved
App\Providers\RouteServiceProvider.php — Dynamic Rate Limiting (v8.0)
use Illuminate\Cache\RateLimiting\Limit;
use Illuminate\Support\Facades\RateLimiter;

protected function configureRateLimiting(): void
{
    RateLimiter::for('api', function (Request $request) {
        // Dynamic limit based on user plan
        return Limit::perMinute(
            $request->user()?->is_premium ? 1000 : 60
        )->by($request->user()?->id ?: $request->ip());
    });
}

// Apply in routes/api.php
Route::middleware(['auth:sanctum', 'throttle:api'])->group(function () {
    Route::apiResource('posts', PostController::class);
});
routes/api.php — Route Controller Grouping (v8.0)
// Before v8.0 — repetitive controller class references
Route::get('/posts',        [PostController::class, 'index']);
Route::post('/posts',       [PostController::class, 'store']);
Route::get('/posts/{post}', [PostController::class, 'show']);

// v8.0+ — cleaner grouping by controller
Route::controller(PostController::class)->group(function () {
    Route::get('/posts',          'index');
    Route::post('/posts',         'store');
    Route::get('/posts/{post}',   'show');
    Route::put('/posts/{post}',   'update');
    Route::delete('/posts/{post}','destroy');
});
database/factories/PostFactory.php — Class-Based Factories for API Testing (v8.0)
class PostFactory extends Factory
{
    public function definition(): array
    {
        return [
            'title'      => $this->faker->sentence,
            'body'       => $this->faker->paragraphs(3, true),
            'user_id'    => User::factory(),
            'published'  => false,
        ];
    }

    // Factory state for published posts
    public function published(): static
    {
        return $this->state(fn () => ['published' => true]);
    }
}

v9.0

Laravel 9.0 — Enums, Scoped Bindings & Symfony 6

February 2022
🔢

PHP 8.1 Enum Route Binding

Route model binding natively supports PHP 8.1 backed enums. Pass an enum value in the URL and Laravel resolves it automatically, returning 404 for invalid values.

New
🔗

Scoped Route Model Binding

Automatically scope nested resource routes. /users/{user}/posts/{post} ensures the post belongs to the user, eliminating manual ownership checks in API controllers.

New
📨

Symfony Mailer Upgrade

Replaced SwiftMailer with Symfony Mailer, bringing modern email transport support. Important for APIs that trigger transactional email notifications.

Breaking Change
🗃️

Implicit Enum Casting in Models

Eloquent models can now cast attributes to PHP enums. API responses automatically serialize enums to their string/integer values.

New
routes/api.php — Enum Route Binding (v9.0)
// Define a PHP 8.1 backed enum
enum Status: string
{
    case Active   = 'active';
    case Inactive = 'inactive';
    case Banned   = 'banned';
}

// Route with enum binding — invalid values auto-return 404
Route::get('/users/status/{status}', function (Status $status) {
    return User::where('status', $status->value)->paginate();
});

// GET /api/users/status/active  -> 200 OK with users
// GET /api/users/status/unknown -> 404 Not Found (automatic)
routes/api.php — Scoped Route Bindings (v9.0)
// Automatically ensures {comment} belongs to {post}
// No need for manual $post->comments()->findOrFail($comment) checks
Route::scopeBindings()->group(function () {
    Route::get('/posts/{post}/comments/{comment}', [
        CommentController::class, 'show'
    ]);
    Route::put('/posts/{post}/comments/{comment}', [
        CommentController::class, 'update'
    ]);
});

// GET /api/posts/5/comments/99
// -> 404 if comment 99 doesn't belong to post 5
// -> 200 if it does

First-party feature flag package. Roll out new API endpoints to a percentage of users, specific users, or based on custom logic — without deploying separate code branches.

All Laravel framework code now uses native PHP type declarations throughout. Better IDE autocompletion, static analysis support, and safer API controller development.

New Process facade for running shell commands. Useful in API workflows that trigger CLI tools or external processes, with full fake/test support built in.

Test Profiling

php artisan test --profile reveals the slowest tests. Essential for keeping large API integration test suites fast as they grow.

New
App\Providers\AppServiceProvider.php — Pennant Feature Flags (v10.0)
use Laravel\Pennant\Feature;

// Define feature flags in AppServiceProvider
public function boot(): void
{
    // 20% rollout of new recommendations engine
    Feature::define('new-recommendations-engine', fn (User $user) =>
        $user->is_beta_tester || random_int(1, 100) <= 20
    );

    // Always active for premium users
    Feature::define('advanced-analytics', fn (User $user) =>
        $user->subscription === 'premium'
    );
}

// In an API controller:
public function recommendations(Request $request): JsonResponse
{
    if (Feature::active('new-recommendations-engine')) {
        return response()->json(
            NewRecommendationService::for($request->user())
        );
    }
    return response()->json(
        LegacyRecommendationService::for($request->user())
    );
}
v11.0

Laravel 11.0 — The Slim API Structure Revolution

March 2024
Biggest API Change Yet: Laravel 11 completely reimagined how API apps are structured. routes/api.php no longer exists by default — it is created on demand via artisan. Sanctum is also opt-in. This is the most significant structural API change since Laravel 5.
📂

Opt-In API Routing

routes/api.php is no longer present in new projects. Run php artisan install:api to scaffold it along with Sanctum configuration automatically.

New Projects
⚡

Per-Second Rate Limiting

Rate limiters now support per-second granularity with perSecond() — critical for high-throughput API endpoints that need burst protection alongside per-minute limits.

New
🔄

Revamped bootstrap/app.php

All routing, middleware, exception handling, and service providers now configured in a single file, replacing the scattered configuration of previous versions.

Redesigned
🏥

Health Check Endpoint /up

Built-in GET /up route for monitoring services, load balancer health checks, and uptime monitoring — no custom route or controller required.

New
🔑

Encryption Key Rotation

Define previous encryption keys alongside the current key, allowing zero-downtime key rotation for encrypted API tokens and cookies.

New
🧪

once() Memoization Helper

New once() helper memoizes the result of a closure for the lifetime of a request. Ideal for expensive operations called multiple times in an API response pipeline.

New
terminal — Setting up API in a new Laravel 11 project
# Create new Laravel 11 project
composer create-project laravel/laravel my-api

# Install API scaffolding (all-in-one command)
php artisan install:api

# This single command:
#  - Creates routes/api.php
#  - Registers api.php in bootstrap/app.php
#  - Installs and configures Laravel Sanctum
#  - Runs Sanctum migrations (personal_access_tokens table)
#  - Reminds you to add HasApiTokens to your User model

# Install broadcasting separately if needed
php artisan install:broadcasting
bootstrap/app.php — New centralized app configuration (v11.0)
use Illuminate\Foundation\Application;
use Illuminate\Foundation\Configuration\Exceptions;
use Illuminate\Foundation\Configuration\Middleware;

return Application::configure(basePath: dirname(__DIR__))
    ->withRouting(
        web:    __DIR__.'/../routes/web.php',
        api:    __DIR__.'/../routes/api.php', // Added by install:api
        health: '/up',                         // Built-in health endpoint
    )
    ->withMiddleware(function (Middleware $middleware) {
        // Append custom middleware to API group
        $middleware->api(append: [
            EnsureApiVersion::class,
            ForceJsonResponse::class,
        ]);
    })
    ->withExceptions(function (Exceptions $exceptions) {
        // Return JSON for all API routes automatically
        $exceptions->shouldRenderJsonWhen(
            fn (Request $r) => $r->is('api/*') || $r->wantsJson()
        );
    })
    ->create();
App\Providers\AppServiceProvider.php — Per-Second Rate Limiting (v11.0)
use Illuminate\Cache\RateLimiting\Limit;
use Illuminate\Support\Facades\RateLimiter;

public function boot(): void
{
    RateLimiter::for('api', function (Request $request) {
        return [
            Limit::perSecond(10),     // Burst protection - NEW in v11
            Limit::perMinute(200)    // Sustained throughput limit
                ->by($request->user()?->id ?: $request->ip()),
        ];
    });
}

Laravel 12.0 — Modern Starter Kits & WorkOS Auth

February 2025
Stability-Focused Release: Laravel 12 is intentionally a low-breaking-change release. Most applications can upgrade with zero code changes. The focus was on dependency upgrades and shipping new modern starter kits.
🎨

New Starter Kits (React, Vue, Livewire)

Completely rebuilt starter kits for React, Vue, and Livewire. Each ships with full authentication scaffolding and API integration out of the box, replacing Breeze and Jetstream as the recommended starting point.

New
🔐

Optional WorkOS variant of starter kits provides social authentication, passkeys, and enterprise SSO — a major upgrade for APIs requiring enterprise-grade auth without building it from scratch.

New
🔄

Breeze & Jetstream Maintenance Mode

Breeze and Jetstream enter maintenance-only status. All new feature development moved to the new starter kits. Existing projects using Breeze/Jetstream continue to work normally.

Decision Guide — Which authentication to use?
Does your API need to allow THIRD-PARTY apps to act on behalf of users?
    YES -> Use Laravel Passport (OAuth2 Authorization Code Grant)
    NO  |

Is it a first-party SPA (Vue, React, Next.js) on same/subdomain?
    YES -> Use Laravel Sanctum (cookie-based session auth + CSRF)
    NO  |

Is it a mobile app or API consumer needing long-lived tokens?
    YES -> Use Laravel Sanctum (personal access tokens)
    NO  |

Simple internal tool with low security requirements?
    YES -> Token Guard works, but consider Sanctum anyway

Modern API Best Practices (Laravel 11+)

A reference for building production-grade APIs incorporating all patterns introduced across versions 5.8 through 12.

App\Http\Resources\PostResource.php — API Resources (best practice, all versions)
namespace App\Http\Resources;

use Illuminate\Http\Resources\Json\JsonResource;

class PostResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id'         => $this->id,
            'title'      => $this->title,
            'body'       => $this->body,
            'status'     => $this->status->value, // Enum serialization (v9+)
            'author'     => new UserResource($this->whenLoaded('user')),
            'tags'       => TagResource::collection($this->whenLoaded('tags')),
            'created_at' => $this->created_at->toISOString(),
            // Conditionally include admin-only fields
            $this->mergeWhen($request->user()?->is_admin, [
                'internal_notes' => $this->internal_notes,
                'ip_address'     => $this->author_ip,
            ]),
        ];
    }
}
App\Http\Controllers\Api\PostController.php — Complete Modern API Controller
namespace App\Http\Controllers\Api;

use App\Http\Requests\StorePostRequest;
use App\Http\Requests\UpdatePostRequest;
use App\Http\Resources\PostResource;
use App\Models\Post;
use Illuminate\Http\Resources\Json\AnonymousResourceCollection;

class PostController extends Controller
{
    public function index(): AnonymousResourceCollection
    {
        $posts = Post::with(['user', 'tags'])    // Eager load to avoid N+1
            ->latest()
            ->paginate(15);                    // Always paginate API lists
        return PostResource::collection($posts);
    }

    public function store(StorePostRequest $request): PostResource
    {
        // Form Request handles all validation
        $post = $request->user()->posts()->create($request->validated());
        return (new PostResource($post))
            ->response()
            ->setStatusCode(201);             // 201 Created
    }

    public function show(Post $post): PostResource
    {
        return new PostResource($post->load(['user', 'tags']));
    }

    public function update(UpdatePostRequest $request, Post $post): PostResource
    {
        $post->update($request->validated());
        return new PostResource($post->fresh(['user', 'tags']));
    }

    public function destroy(Post $post): \Illuminate\Http\Response
    {
        $this->authorize('delete', $post);     // Policy authorization
        $post->delete();
        return response()->noContent();       // 204 No Content
    }
}
routes/api.php — API Versioning Best Practice
use Illuminate\Support\Facades\Route;

// Version 1 API
Route::prefix('v1')->name('v1.')->group(function () {
    Route::middleware('auth:sanctum')->group(function () {
        Route::apiResource('posts',    V1\PostController::class);
        Route::apiResource('comments', V1\CommentController::class);

    // Public routes (no auth)

Route::prefix('v2')->name('v2.')->group(function () {
    Route::middleware('auth:sanctum')->group(function () {

        Route::apiResource('reactions', V2\ReactionController::class);
    });
});
