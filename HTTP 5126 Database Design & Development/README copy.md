# Database Design
># 📘 What the Database Design Course Covers

The Database Design course gives students a solid foundation in how modern databases are structured and used in real‑world environments. You’ll learn the principles of relational database design — including creating entity‑relationship diagrams (ERDs) and applying normalization techniques to organize data efficiently — then take those designs and implement them using a database language like SQL on industry‑standard systems such as Oracle or MySQL. The focus is on both theory and hands‑on practice, so you’ll not only understand why good design matters, but also how to build a working database that correctly stores, retrieves, and manipulates data.

># 🛠️ Skills and Practical Experience

In this course, you’ll get practical experience with data modelling tools and SQL commands to create tables, write queries, and work with single and multiple tables — including using functions and subqueries to solve more advanced data retrieval tasks. By the end of the term, you should be comfortable translating real‑world requirements into logical database structures and writing SQL to implement them, as well as understanding how applications connect to and interact with databases. This prepares you well for roles that involve database administration, application development, or further studies in software and systems design.
